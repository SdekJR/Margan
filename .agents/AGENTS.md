# Project Rules

- The user communicates in Russian.
- All documentation (including artifacts, plans, comments, and project documentation) must be written in English.
- Currently, the project is in the specification stage. All documents must be Markdown (`.md`) files focusing on use cases, user scenarios, and functional requirements (answering *what* the system does and *why*), rather than technical architecture or software implementation details (answering *how*).
- Any web research conducted to supplement or verify the use cases must be summarized and presented to the user for review and approval before being integrated into the project's specification documents.
